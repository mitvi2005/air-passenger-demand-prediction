About AirPax

A machine learning research project predicting airline passenger demand using 21 model variants across 6 algorithm families, with a live self-retraining Flask API.

Project Overview

AirPax forecasts monthly airline passenger volume across 1949–2026 using 21 model variants. The best model, Linear Regression, currently reports R² = {{ '%.4f'|format(stats.r2 if stats else 0) }} on month and year features.

Every prediction triggers a self-retraining loop : new data is appended and the model retrains automatically, so R², MAE, MSE, RMSE, and total row count stay live across the UI.

Why Linear Regression Won
→Passenger growth follows a near-perfect linear time trend
→Only 2 features, so the model stays simple and interpretable
→Boosting and tree models can overfit on this lightweight feature space
→The retraining loop keeps the latest dataset reflected in the live metrics

Development Timeline
Phase 1
Data Collection & EDA
Extended dataset 1949–2026. Trend visualization and seasonality analysis with seaborn and matplotlib.
Phase 2
Feature Engineering
Extracted month + year. IQR outlier removal, StandardScaler, PolynomialFeatures for linear variants.
Phase 3
Model Training (21 Variants)
Linear, Ridge, Lasso, ElasticNet, Decision Tree, Random Forest, SVR, KNN, XGBoost, GradientBoosting, AdaBoost with simple and tuned variants.
Phase 4
Flask API + Self-Retraining
Each prediction updates the dataset and triggers automatic model retrain, which refreshes rows and all live metrics.
Phase 5
Premium Dashboard UI
Interactive charts, multi-page navigation, modal popups, and a live metric dashboard experience.
